Lo que vamos a ver también existe en React, Vue, Angular y prácticamente cualquier framework moderno: **componentizar la interfaz**.

## 1. El problema

Si tenemos todas estas páginas:

```

home.pug
about.pug
contact.pug
login.pug
products.pug

```

Todas necesitan mostrar el mismo encabezado.

``` html

<header>
    <img src="/logo.png">

    <nav>
        Home
        Products
        Contact
        Login
    </nav>
</header>

```

Si copiamos ese código en los cinco archivos:

```

home.pug
  header...

about.pug
  header...

contact.pug
  header...

login.pug
  header...

products.pug
  header...

```

- Tenemos un problema.

### ¿Qué pasa si mañana cambiamos el menú?

Por ejemplo agregamos:

- Blog

Ahora debemos modificar:

- home.pug

- about.pug

- contact.pug

- login.pug

- products.pug

Cinco archivos.

Si fueran 40 páginas, tendríamos que editar 40 archivos.

## 2. La solución

Creamos un archivo llamado

```

_header.pug

```

Con el código:

``` pug

header
    img(src="/logo.png")

    nav
        ul
            li Home
            li Products
            li Contact
            li Login

```

Ahora todas las páginas simplemente hacen:

``` pug

include _header

```

## 3. Entonces `base.pug` queda así

``` pug

doctype html
html
    head
        title Mi sitio

    body

        include _header

        main
            h1 Bienvenido

        include _footer

```

Queda más limpio el código.

## 4. ¿Qué hace `include`?

Cuando Pug encuentra

``` pug

include _header

```

Es como si copiara el contenido de `_header.pug` y lo pegara ahí antes de generar el HTML.

Es decir, internamente hace algo parecido a esto:

Antes:

``` pug

body

    include _header

    main

```

Después de procesarlo:

``` pug

body

    header
        img(...)
        nav
            ...

    main

```

Finalmente genera el HTML.

## 5. ¿Por qué el archivo empieza con "_"?

No es obligatorio.

Pero es una convención.

El guion bajo indica:

- Este archivo no es una página completa.

Es solamente un fragmento reutilizable.

Algo parecido a:

```

-_header.pug
_footer.pug
_navigation.pug
_sidebar.pug

```
