# Y ahora `mixin`

Reutiliza un fragmento de código que puede recibir parámetros, como una función.

## 1. ¿Qué problema resuelven los mixins?

Por ejemplo tenemos una lista de productos.

- iPhone

- Samsung

- Pixel

- Xiaomi

Cada producto se muestra igual:

``` html

<div class="producto">

    <h2>iPhone</h2>

    <p>$999</p>

</div>

```

Y después:

``` html

<div class="producto">

    <h2>Samsung</h2>

    <p>$899</p>

</div>

```

Estamos copiando la misma estructura una y otra vez.

### Sin mixins

En Pug escribiríamos algo como:

``` pug

div.producto
    h2 iPhone
    p $999

div.producto
    h2 Samsung
    p $899

div.producto
    h2 Pixel
    p $799

```

La única diferencia son los datos.

## 2. Con un mixin

Primero definimos una función:

``` pug

mixin producto(nombre, precio)

    div.producto
        h2= nombre
        p= precio

```

**Esto se parece muchísimo a JavaScript.**

### Ahora lo llamamos

``` pug

+producto("iPhone", "$999")

+producto("Samsung", "$899")

+producto("Pixel", "$799")

```

Cuando Pug renderiza la página, obtiene:

``` pug

<div class="producto">

<h2>iPhone</h2>

<p>$999</p>

</div>

<div class="producto">

<h2>Samsung</h2>

<p>$899</p>

</div>

<div class="producto">

<h2>Pixel</h2>

<p>$799</p>

</div>

```

## 3. ¿Por qué el signo `+`?

Cuando vemos

``` pug

+producto(...)

```

Significa:

- Ejecuta este mixin.

Es igual que en JavaScript hacer:

``` javascript

producto("iPhone", "$999");

```
## 4. Otro ejemplo

Supongamos que todo el sitio tiene botones iguales.

#### Sin mixins:

``` pug

button.btn.btn-primary Guardar

button.btn.btn-primary Editar

button.btn.btn-primary Eliminar

```

Repetimos las clases todo el tiempo.

#### Con un mixin:

``` pug

mixin boton(texto)

    button.btn.btn-primary= texto

```

Ahora:

``` pug

+boton("Guardar")

+boton("Editar")

+boton("Eliminar")

```

Muchísimo más limpio.

## 5. Incluso puede tener lógica

``` pug

mixin boton(texto, activo)

    if activo

        button.btn.btn-primary= texto

    else

        button.btn.btn-secondary= texto

```

Luego

``` pug

+boton("Guardar", true)

+boton("Eliminar", false)

```

Obtenemos dos botones diferentes usando el mismo mixin.

## 6. ¿Dónde suelen guardarse?

Normalmente se crea un archivo:

- mixins.pug

o

- _mixins.pug

Dentro:

``` pug

mixin boton(...)

mixin producto(...)

mixin usuario(...)

```

Y luego en otra plantilla:

``` pug

include _mixins

```

Así todos los mixins están disponibles para usar.

## 7. ¿En qué se diferencia de include?

Esta es la duda más común.

### Include

Inserta siempre el mismo contenido.

``` pug

include _header

```

Siempre aparece exactamente el mismo encabezado.

### Mixin

Genera contenido diferente según los parámetros.

``` pug

+boton("Guardar")

+boton("Eliminar")

+boton("Cancelar")

```

Es el mismo "molde", pero con datos distintos.

## Resumen

| Concepto            | ¿Qué reutiliza?                            | Equivalente aproximado                                         |
| ------------------- | ------------------------------------------ | -------------------------------------------------------------- |
| `include`           | Un archivo completo (header, footer, menú) | Copiar y pegar automáticamente un archivo                      |
| `extends` + `block` | La estructura base de una página           | Una plantilla con espacios que otras páginas rellenan          |
| `mixin`             | Un bloque reutilizable que recibe datos    | Una función en JavaScript o un componente con `props` en React |
