# Herencia de Plantillas

Pensemos en un sitio web

Por ejemplo tenemos un sitio con estas páginas:

```

Inicio
Productos
Contacto
Nosotros
Login

```

Todos tienen la misma estructura.

```

-------------------------------------

LOGO

MENÚ

-------------------------------------

(Aquí cambia el contenido)

-------------------------------------

FOOTER

-------------------------------------

```

## 1. Sin extends

Podríamos hacer esto:

```

home.pug

```

``` pug

doctype html

html

    head
        title Inicio

    body

        include _header

        h1 Bienvenido

        include _footer

```

Luego:

```

contact.pug

```

``` pug

doctype html

html

    head
        title Contacto

    body

        include _header

        h1 Contáctanos

        include _footer

```

El patrón es el mismo, se repite:

- doctype

- html

- head

- body

- include header

- include footer

En todas las páginas.

## 2. Aquí entra extends

En lugar de repetir esa estructura una y otra vez, creamos un archivo llamado.

- base.pug

Ese archivo representa el esqueleto de todas las páginas.

``` pug

doctype html

html

    head

        title Mi sitio

    body

        include _header

        block contenido

        include _footer

```

Línea importante:

- block contenido

Eso significa:

- "Aquí irá el contenido específico de cada página."

Todavía no sabemos cuál será.

Solo dejamos el espacio reservado.

## Supongamos que hacemos

- home.pug

El archivo ya no necesita escribir todo.

Solo dice:

``` pug

extends base

block contenido

    h1 Bienvenido

    p Esta es la página principal.

```

Eso es todo.

No hay:

- html

- body

- header

- footer

Porque ya existen en base.pug.

## ¿Qué hace Pug internamente?

Cuando encuentra

``` pug

extends base

```

Hace algo parecido a esto:

Toma:

- base.pug

``` pug

doctype html

html

    head

        title Mi sitio

    body

        include _header

        block contenido

        include _footer

```

Y toma

- home.pug

``` pug

block contenido

    h1 Bienvenido

    p Hola

```

Los mezcla.

El resultado final es

``` html

<!DOCTYPE html>

<html>

<head>

<title>Mi sitio</title>

</head>

<body>

<header>...</header>

<h1>Bienvenido</h1>

<p>Hola</p>

<footer>...</footer>

</body>

</html>

```

## Resumen

### ¿Qué hace block?

Un bloque es simplemente un espacio reservado.

``` pub

block contenido

```

Significa

- "Aquí alguien pondrá contenido."

Nada más.

### ¿Y extends?

Extends significa

- "Quiero usar esta plantilla como base."
