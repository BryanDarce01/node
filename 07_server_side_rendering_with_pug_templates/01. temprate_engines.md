# Pug

Con Pug, Express puede generar páginas HTML dinámicamente.

[app.js](https://github.com/jonasschmedtmann/complete-node-bootcamp/blob/master/4-natours/after-section-12/app.js)

## 1. ¿Qué hace `app.set('view engine', 'pug')`?

``` javascript

app.set('view engine', 'pug');

```

Le dice a Express:

 - El motor de plantillas que voy a usar para generar HTML es Pug.

Antes, cuando hacíamos una petición:

```

GET /api/v1/tours

```

Respondíamoss con:

``` javascript

res.status(200).json({
  status: 'success',
  data: tours
});

```

El navegador recibía JSON.

Ahora, en lugar de JSON, Express puede devolver una página HTML.

## 2. ¿Qué hace `app.set('views', ...)`?

``` javascript

app.set('views', path.join(__dirname, 'views'));

```

Esta línea le dice a Express:

- Las plantillas Pug están dentro de la carpeta `views`.

Ejemplo:

```

project/
│
├── app.js
├── server.js
├── controllers/
├── models/
├── routes/
└── views/
    ├── base.pug
    ├── overview.pug
    └── tour.pug

```

Cuando luego escribimos:

``` javascript

res.render('base');

```

Express busca automáticamente:

```

views/base.pug

```

## ¿Qué hace `path.join()`?

Supongamos que:

``` javascript

__dirname

```

Y la ruta:

```

C:\Users\Bryan\Natours

```

Entonces:

``` javascript

path.join(__dirname, 'views');

```

produce:

```

C:\Users\Bryan\Natours\views

```

- Así Express sabe dónde buscar las plantillas.

## 3. ¿Qué hace `res.render()`?

Ejemplo tenemos:

``` javascript

app.use('/', (req, res) => {
  res.status(200).render('base');
});

```

Cuando visitamos:

```

http://localhost:3000/

```

Express hace lo siguiente:

1. Busca views/base.pug

2. Convierte el archivo Pug en HTML

3. Envía ese HTML al navegador

## 4. ¿Qué es un archivo `.pug`?

Es una forma más corta y limpia de escribir HTML.

### HTML normal

``` html

<!DOCTYPE html>
<html>
  <head>
    <title>Natours</title>
  </head>

  <body>
    <h1>Hola Mundo</h1>
  </body>
</html>

```

### En Pug

``` pug

doctype html

html
  head
    title Natours

  body
    h1 Hola Mundo

```

Mucho más corto porque no necesitamos escribir etiquetas de cierre (`</html>`, `</body>`, etc.).

En Pug, la indentación define la estructura.

## 5. ¿Por qué usar Pug?

Porque permite generar HTML dinámicamente.

Por ejemplo:

``` javascript

res.render('tour', {
  title: 'The Forest Hiker'
});

```

Y en `tour.pug`:

``` pug

h1= title

```

Pug genera:

``` html

<h1>The Forest Hiker</h1>

```

Si cambia el valor:

``` javascript

res.render('tour', {
  title: 'The Sea Explorer'
});

```

El mismo archivo produce:

``` html

<h1>The Sea Explorer</h1>

```

No se necesita crear un HTML diferente para cada tour; una sola plantilla sirve para todos.

## Resumen del flujo

1. Usuario visita:

```

http://localhost:3000/

```

2. Express busca:

```

views/base.pug

```

3. Convierte Pug → HTML

4. Envía el HTML al navegador
