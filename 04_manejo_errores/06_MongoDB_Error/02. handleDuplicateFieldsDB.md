# `handleDuplicateFieldsDB` (error de campo duplicado)

``` javascript

const handleDuplicateFieldsDB = err => {
  const value = err.errmsg.match(/(["'])(\\?.)*?\1/)[0];

  const message = `Duplicate field value: ${value}. Please use another value!`;
  return new AppError(message, 400);
};

```

## ¿Qué error está manejando?

Este error ocurre cuando tenemos un campo con `unique: true` en Mongoose:

``` javascript

email: {
  type: String,
  unique: true
}

```

## Ejemplo

Intentamos crear:

``` json

{
  "email": "test@test.com"
}

```

- Pero ese email ya existe.

## 🔴 Mongo lanza algo así:

```

E11000 duplicate key error collection: users index: email_1 dup key: { email: "test@test.com" }

```

Este error tiene:

``` javascript

{
  code: 11000,
  errmsg: 'E11000 duplicate key error ... "test@test.com"'
}

```

## ¿Qué hace la función?

## 1. Extraer el valor duplicado

``` javascript

const value = err.errmsg.match(/(["'])(\\?.)*?\1/)[0];

```

- Usa una **expresión regular (regex)**

- El `[0]` está ahí porque `.match()` devuelve un **array**.

### ¿Qué hace el regex?

```

/(["'])(\\?.)*?\1/

```

Busca:

- texto entre comillas `"..."` o `'...'`

### ¿Qué devuelve `.match()`?

Cuando hacemos:

``` javascript

err.errmsg.match(/(["'])(\\?.)*?\1/)

```

Obtenemos algo así:

``` javascript

[
  '"test@test.com"', // ← posición 0 (match completo)
  '"',               // grupo capturado 1
  'm',               // parte interna (depende del match)
  ...
]

```

Entonces:

``` javascript

[0]

```

Significa:

- **Dame el texto completo que hizo match**

De esto:

```

E11000 duplicate key error ... { email: "test@test.com" }

```
A esto =>

``` javascript

[
  '"test@test.com"',
  ...
]

```

### Resultado

```

"test@test.com"

```

## 2. Crear mensaje limpio

``` javascript

const message = `Duplicate field value: ${value}. Please use another value!`;

```

Resultado:

```

Duplicate field value: "test@test.com". Please use another value!

```

## 3. Convertir a AppError

``` javascript

return new AppError(message, 400);

```

Ahora es:

- error operacional ✅

- status 400 (bad request)

## Resultado

``` json

{
  "status": "fail",
  "message": "Duplicate field value: \"test@test.com\". Please use another value!"
}

```

## Idea clave

Estamos pasando de esto:

```

E11000 duplicate key error collection...

```

A esto:

```

Duplicate field value: "test@test.com"

```

- Claro

- limpio
