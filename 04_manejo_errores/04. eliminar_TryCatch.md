# `catchAsync`

Es un patrón muy común en **Express** para evitar repetir `try/catch` en cada controlador `async`.

## El problema que resuelve

En Express, si tenemos algo así:

``` javascript

app.get('/users', async (req, res, next) => {
  try {
    const users = await User.find();
    res.json(users);
  } catch (err) {
    next(err);
  }
});

```

- Tenemos que escribir `try/catch` en **cada ruta async**, lo cual es repetitivo.

## La solución

``` javascript

const catchAsync = fn => {

    return (req, res, next) => {
        fn(req, res, next).catch(next)
    };
};

```

## Pasos

### 1. Recibe una función (fn)
Esa función es el controlador **async**

``` javascript

async (req, res, next) => {
  const users = await User.find();
  res.json(users);
}

```

### 2. Devuelve otra función

- middleware de Express

``` javascript

(req, res, next) => { ... }

```

- Esto es lo que Express realmente ejecuta.

### 3. Ejecuta la función y captura errores automáticamente

``` javascript

fn(req, res, next).catch(next);

```

Clave:

- Si `fn` falla (throw o error en await)

- `.catch(next)` manda el error a Express

- Express lo envía al error handler global

## Cómo se usa

En vez de esto:

``` javascript

app.get('/users', async (req, res, next) => {
  try {
    const users = await User.find();
    res.json(users);
  } catch (err) {
    next(err);
  }
});

```

Usamos esto:

``` javascript

app.get('/users', catchAsync(async (req, res, next) => {
  const users = await User.find();
  res.json(users);
}));

```

- ✅ Sin try/catch

- ✅ Más limpio

- ✅ Todos los errores van al mismo lugar

## Conexión con el `globalErrorHandler`

Cuando hacemos:

``` javascript

.catch(next);

```

Es lo mismo que hacer:

``` javascript

next(error);

```

Y eso activa este middleware:

``` javascript

app.use((err, req, res, next) => {
  res.status(500).json({
    message: err.message
  });
});

```

## Resumen

- `catchAsync` = envoltorio (wrapper)

- Ejecuta funciones **async**

- Si hay error → lo manda a `next`

- Así evitamos repetir `try/catch`
