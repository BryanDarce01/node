# CORS significa:

- Cross-Origin Resource Sharing

## El problema que resuelve

Los navegadores tienen una regla de seguridad:

- Un frontend NO puede hacer requests a cualquier dominio libremente.

Ejemplo

### Frontend:

```

http://localhost:3000

```
### Backend:

```

http://localhost:8000

```

Aunque ambos estén en la misma PC:

❌ son origins diferentes.

Porque cambia:

- puerto

- dominio

- protocolo

## 1. ¿Qué es un ORIGIN?

Un origin es:

```

PROTOCOLO + DOMINIO + PUERTO

```

## Cambia cualquiera → origin diferente

| URL                                                    | Diferente |
| ------------------------------------------------------ | --------- |
| [http://localhost:3000](http://localhost:3000)         | puerto    |
| [https://localhost:3000](https://localhost:3000)       | protocolo |
| [http://api.localhost:3000](http://api.localhost:3000) | dominio   |

## Entonces el navegador bloquea

Cuando React intenta:

``` javascript

fetch('http://localhost:8000/api')

```

el navegador dice:

- “¿El backend permite esto?”

## El backend responde con headers CORS

Ejemplo:

``` javascript

Access-Control-Allow-Origin: http://localhost:3000

```

Entonces el navegador permite la request.

## Importante

CORS es:

✅ seguridad del navegador

NO:

- Express

- Node

- JWT

## MUY importante

Postman NO respeta CORS.

Porque:

- CORS solo existe en navegadores.

Por eso:

- Postman funciona

- React falla

y la gente se confunde muchísimo.

## En Express normalmente hacemos:

``` bash

npm install cors

```

``` javascript

const cors = require('cors');

app.use(cors());

```

Eso permite:

```

Access-Control-Allow-Origin: *

```

Pero en producción normalmente solo permitimos las url´s (de los navegadores) que queremos, haga solicitudes al backend:

``` javascript

app.use(cors({
  origin: 'https://myfrontend.com'
}));

```

## Qué requests activan más CORS

Especialmente:

- Authorization headers

- JSON

- métodos PUT/PATCH/DELETE

## Preflight Request (MUY importante)

A veces el navegador primero manda:

```

OPTIONS /api/users

```

antes de la request real.

### ¿Por qué?

Pregunta:

- “Backend, ¿aceptas este tipo de request?”

El backend responde:

```

Access-Control-Allow-Methods
Access-Control-Allow-Headers

```

Y luego el navegador manda la request real.
