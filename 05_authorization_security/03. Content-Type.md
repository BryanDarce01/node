# ¿Qué es `Content-Type`?

Es un header HTTP que le dice al servidor:

- “El contenido que te estoy enviando tiene este formato”

## ¿Por qué importa?

Porque el servidor necesita saber:

- cómo interpretar los datos

- cómo parsearlos

- qué hacer con ellos

### Ejemplo con JSON

``` http

Content-Type: application/json

```

Body:

``` json

{
  "email": "test@test.com"
}

```

En Express:

``` javascript

app.use(express.json());

```

- `express.json()` solo funciona para:

``` http

application/json

```

## Otros tipos importantes de Content-Type

### 🟢 1. `application/json`

El más usado.

Ejemplo

``` json

{
  "name": "Jonas"
}

```

**Uso típico**

- APIs REST

- frontend/backend

- mobile apps

### 🟡 2. `application/x-www-form-urlencoded`

Formato clásico de formularios HTML.

Ejemplo:

```

name=Jonas&age=30

```

**Lo generan formularios HTML tradicionales**

``` html

<form method="POST">

```

**En Express**

``` javascript

app.use(express.urlencoded({ extended: true }));

```


### 🔴 3. `multipart/form-data`

Se usa para:

- subir imágenes

- archivos

- PDFs

- videos

Ejemplo

``` http

Content-Type: multipart/form-data

```

Aquí los datos vienen separados por “partes”

Una parte puede ser:

- texto

- imagen

- archivo

En Express normalmente se usa:

``` javascript

multer

```

### 🟣 4. `text/plain`

Texto plano.

Ejemplo

```

Hola mundo

```

### 🔵 5. application/xml

Antes era MUY usado.

Ejemplo

``` xml

<user>
  <name>Jonas</name>
</user>

```

**Hoy JSON domina casi todo.**

- “¿Qué formato tienen los datos?”
