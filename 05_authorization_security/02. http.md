# 🌐 ¿Qué es HTTP?

HTTP significa:

- **HyperText Transfer Protocol**

Es el protocolo que usan:

- navegadores

- apps

- frontend

- backend

- APIs

para comunicarse.

En otras palabras:

- **Es un sistema de mensajes entre cliente y servidor.**

## TODO gira alrededor de:

### 📤 REQUEST

Lo que envía el cliente.

### 📥 RESPONSE

Lo que responde el servidor.

# REQUEST HTTP

Ejemplo:

``` http
POST /api/v1/users/login HTTP/1.1
Host: myapp.com
Content-Type: application/json
Authorization: Bearer abc123

{
  "email": "test@test.com",
  "password": "123456"
}

```

Vamos por partes.

## 1. METHOD (GET, POST, etc.)

La primera palabra:

``` http

POST

```

- Indica lo qué queremos hacer.

### Métodos importantes

| Método | Acción                  |
| ------ | ----------------------- |
| GET    | Obtener datos           |
| POST   | Crear                   |
| PATCH  | Actualizar parcialmente |
| PUT    | Reemplazar              |
| DELETE | Eliminar                |

### Ejemplos

**1. Obtener usuarios**

```

GET /users

```

**2. Crear usuario**

```

POST /users

```

**3. Actualizar email**

```

PATCH /users/123

```

## 2. URL / ROUTE

``` http

/api/v1/users/login

```

- Es el recurso al que queremos acceder.

En Express:

``` javascript

app.post('/api/v1/users/login', loginController);

```

## 3. HEADERS

Los headers son:

- metadata de la request

No son los datos principales.

### Headers comunes

| Header        | Para qué sirve           |
| ------------- | ------------------------ |
| Content-Type  | Qué tipo de datos envías |
| Authorization | Autenticación            |
| Cookie        | Cookies                  |
| Accept        | Qué formatos aceptas     |

### Content-Type

Ejemplo:

``` http

Content-Type: application/json

```

Le dice al servidor:

- “Te estoy mandando JSON”

Sin eso:

``` javascript

req.body

```

Muchas veces no funcionaría correctamente.

### Authorization

``` http

Authorization: Bearer eyJhbGc...

```

- Aquí mandamos credenciales.

### Bearer

Significa:

- “el portador del token”

El token puede ser:

- JWT

- API key

- otros

## 4. BODY

El body contiene:

- los datos principales de la petición

Ejemplo

``` json

{
  "email": "jonas@test.com",
  "password": "123456"
}

```

### Cuándo usar body

Principalmente:

- POST

- PATCH

- PUT

En Express

``` javascript

req.body

```

## 5. QUERY PARAMETERS

Van en la URL:

``` http

GET /users?page=2&limit=10

```
