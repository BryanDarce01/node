# 🌐 ¿Qué es HTTP?

HTTP significa:

- **HyperText Transfer Protocol**

Es el protocolo que usan:

- navegadores

- apps

- frontend

- backend

- APIs

para comunicarse.

En otras palabras:

- **Es un sistema de mensajes entre cliente y servidor.**

## Todo gira alrededor de:

### 📤 REQUEST

Lo que envía el cliente.

### 📥 RESPONSE

Lo que responde el servidor.

# REQUEST HTTP

Ejemplo:

``` http
POST /api/v1/users/login HTTP/1.1
Host: myapp.com
Content-Type: application/json
Authorization: Bearer abc123

{
  "email": "test@test.com",
  "password": "123456"
}

```

Vamos por partes.

## 1. METHOD (GET, POST, etc.)

La primera palabra:

``` http

POST

```

- Indica lo qué queremos hacer.

### Métodos importantes

| Método | Acción                  |
| ------ | ----------------------- |
| GET    | Obtener datos           |
| POST   | Crear                   |
| PATCH  | Actualizar parcialmente |
| PUT    | Reemplazar              |
| DELETE | Eliminar                |

### Ejemplos

**1. Obtener usuarios**

```

GET /users

```

**2. Crear usuario**

```

POST /users

```

**3. Actualizar email**

```

PATCH /users/123

```

## 2. URL / ROUTE

``` http

/api/v1/users/login

```

- Es el recurso al que queremos acceder.

En Express:

``` javascript

app.post('/api/v1/users/login', loginController);

```

## 3. HEADERS

Los headers son:

- metadata de la **request**

No son los datos principales.

### Headers comunes

| Header        | Para qué sirve           |
| ------------- | ------------------------ |
| Content-Type  | Qué tipo de datos enviamos |
| Authorization | Autenticación            |
| Cookie        | Cookies                  |
| Accept        | Qué formatos acepta     |

### [Content-Type](03.%20Content-Type.md)

Ejemplo:

``` http

Content-Type: application/json

```

Le dice al servidor:

- “Te estoy mandando JSON”

Sin eso:

``` javascript

req.body

```

Muchas veces no funcionaría correctamente.

### Authorization

``` http

Authorization: Bearer eyJhbGc...

```

- Aquí mandamos credenciales.

### Bearer

Significa:

- “el portador del token”

El token puede ser:

- JWT

- API key

- otros

## 4. BODY

El body contiene:

- los datos principales de la petición

Ejemplo

``` json

{
  "email": "jonas@test.com",
  "password": "123456"
}

```

### Cuándo usar body

Principalmente:

- POST

- PATCH

- PUT

En Express

``` javascript

req.body

```

## 5. QUERY PARAMETERS

Van en la URL:

``` http

GET /users?page=2&limit=10

```

### En Express

``` javascript

req.query

```

Resultado:

``` javascript

{
  page: '2',
  limit: '10'
}

```

## 6. ROUTE PARAMS

Ejemplo:

``` http

GET /users/123
En Express
req.params

```

Resultado:

``` javascript

{
  id: '123'
}

```

## Entonces

| Parte        | Ejemplo       | Express       |
| ------------ | ------------- | ------------- |
| Route params | `/users/123`  | `req.params`  |
| Query params | `?page=2`     | `req.query`   |
| Body         | JSON enviado  | `req.body`    |
| Headers      | Authorization | `req.headers` |

## RESPONSE HTTP

El servidor responde:

``` http

HTTP/1.1 200 OK
Content-Type: application/json

{
  "status": "success"
}

```

## 7. STATUS CODE

Indica qué pasó.

### Códigos importantes

| Código | Significa      |
| ------ | -------------- |
| 200    | OK             |
| 201    | Creado         |
| 400    | Bad Request    |
| 401    | No autenticado |
| 403    | Prohibido      |
| 404    | No encontrado  |
| 500    | Error servidor |

#### Diferencia importante

### 401

- “No has iniciado sesión”

### 403

- “Sí iniciaste sesión, pero no tienes permisos”

## RESPONSE HEADERS

Igual que en request:

- metadata

Ejemplo:

``` http

Content-Type: application/json

```

### RESPONSE BODY

Los datos que responde el servidor:

``` json

{
  "status": "success",
  "data": {
    "user": "Jonas"
  }
}

```

## Cómo Express abstrae HTTP

HTTP puro es complicado.

Express nos da objetos amigables:

### REQUEST

``` javascript

req.body
req.params
req.query
req.headers

```

### RESPONSE

``` javascript

res.status(200).json({...})

```

## Flujo completo

### 📤 Cliente

Envía:

- método

- URL

### 🌐 HTTP

Transporta el mensaje.

### ⚙️ Express

Convierte eso en:

``` javascript

req
res

```

### 🗄️ Backend

Procesa:

- DB

- auth

- lógica

### 📥 Response

Devuelve:

- status code

- headers

- body

## IMPORTANTE

HTTP es solo:

- un protocolo de comunicación.

Todo lo demás:

- JWT

- REST

- APIs

- Express

- frontend

está construido ENCIMA de HTTP.
