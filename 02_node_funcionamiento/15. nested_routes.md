# `mergeParams: true`

Es una opción de **Express.js** que se usa cuando trabajamos con routers anidados (**nested routes**).

El problema que resuelve es que, por defecto, un router hijo **no puede acceder a los parámetros de la URL definidos en el router padre**.

## 1. Sin `mergeParams: true`

Supongamos que tenemos esta ruta:

``` javascript

app.use('/api/v1/tours/:tourId/reviews', reviewRouter);

```

Y dentro de `reviewRouter`:

``` javascript

const router = express.Router();

```

Si hacemos una petición:

``` javascript

GET /api/v1/tours/123/reviews

```

Dentro del controlador:

``` javascript

console.log(req.params);

```

Obtendríamos:

``` javascript

{}

```

- o solamente los parámetros definidos dentro del propio router.

El `tourId` pertenece al router padre y no se hereda automáticamente.

## 2. Con `mergeParams: true`

Creamos el router así:

``` javascript

const router = express.Router({
  mergeParams: true
});

```

Ahora, para la misma petición:

``` javascript

GET /api/v1/tours/123/reviews

```

Tenemos:

``` javascript

req.params

```

``` javascript

{
  tourId: '123'
}

```

Esto es útil porque muchas veces queremos crear recursos "hijos" de otro recurso.

Por ejemplo:

- **A. Reviews de un Tour**

```

/tours/123/reviews

```

- **B. Comentarios de un Post**

```

/posts/456/comments

```

- **C. Órdenes de un Usuario**

```

/users/789/orders

```

El router hijo necesita saber:

- cuál tour

- cuál post

- cuál usuario

Y esa información viene en los parámetros del router padre.

## Ejemplo de creación de review

Ruta:

```

POST /api/v1/tours/123/reviews

```

Controlador:

``` javascript

exports.createReview = async (req, res) => {
  if (!req.body.tour) {
    req.body.tour = req.params.tourId;
  }

  req.body.user = req.user.id;

  const review = await Review.create(req.body);

  res.status(201).json({
    status: 'success',
    data: {
      review
    }
  });
};

```

Aquí `req.params.tourId` solo existe porque el router fue creado con:

``` javascript

express.Router({
  mergeParams: true
});

```

## Resumen

Si tenemos rutas anidadas como:

```

/parent/:id/child

```

y el router de `child` necesita acceder al `:id` del `parent`, entonces usamos:

``` javascript

const router = express.Router({
  mergeParams: true
});

```

Si no hay rutas anidadas o el router hijo no necesita parámetros del padre, no hace falta usarlo.
