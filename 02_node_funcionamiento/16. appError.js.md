# Esta clase se usa para crear errores

## ¿Qué es `AppError`?

`appError` es el nombre que le pongo al archivo que contiene la clase.

``` javascript

class AppError extends Error {
    constructor(message, statusCode){
        super(message)

        this.statusCode = statusCode;
        this.status = `${statusCode}`.startsWith('4') ? 'fail' : 'error';
        this.isOperational = true

        Error.captureStackTrace(this, this.constructor)
    }
}

module.exports = AppError;

```

Aquí estamos creando nuestro propio **tipo de error personalizado**

### En vez de usar:

``` javascript

new Error('algo salió mal')

```
Ahora usamos:

``` javascript

new AppError('algo salió mal', 404)

```

## ¿Por qué hacer esto?

Porque el `Error` normal:

- ❌ no tiene statusCode

- ❌ no diferencia tipos de error

- ❌ no sabe si es operacional o bug

`AppError` soluciona eso

# Desgloce de la clase `AppError`

## Paso 1.

``` javascript

class AppError extends Error

```

- En esta parte lo que estamos haciendo es crear un tipo de error más inteligente basado en el error nativo de JavaScript.

- `Error` ya existe en JS. Aquí lo estamos **extendiendo**.

## 2. El constructor

``` javascript

constructor(message, statusCode)

```
Cuando hacemos esto:

``` javascript

new AppError('No encontrado', 404)

```

- Se ejecuta el constructor.

## 3. ¿Por qué `super(message)`?

## Al crear la clase

``` javascript

class AppError extends Error

```

- No estamos creando un error todavía.

- Solo estamos definiendo una clase.

### Entonces… ¿qué hace `super(message)`?

- Inicializa la parte `Error` de la clase.

``` javascript

super(message)

```

En otras palabras, le decimos a JS:

***Ejecuta el constructor de `Error` y crea correctamente este error”***

- Esto es OBLIGATORIO cuando extendemos una clase.

- Hacemos esto porque `message` es un atributo de `Error` y esta es la forma de inicializarlo.

### ¿Por qué pasa eso?

Porque en JavaScript:

- Cuando usamos `extends`, el objeto **no está listo hasta que llamemos a** `super()`

## Sin `super` pasaría esto

``` javascript

class AppError extends Error {
  constructor(message) {
    this.message = message // ❌ ERROR
  }
}

```

### Con `super(message)`

``` javascript

super(message)

```

Hace esto internamente:

- crea el objeto base

- define `this.message`

- define `this.stack`

- Hace que `instanceof Error` funcione

- inicializa el error correctamente

- Hace que Express lo trate como error real
