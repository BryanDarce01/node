# Ruta `getMe()`

Esta ruta sirve para que el usuario obtenga toda su información.

## ¿Cómo lo hacemos?

Modifica el objeto `req.params` antes de que llegue al siguiente middleware.

``` javascript

exports.getMe = (req, res, next) => {
  req.params.id = req.user.id;
  next();
};

```

Normalmente para obtener un usuario específico tendríamos una ruta como:

```

GET /api/v1/users/648ab123

```

y Express coloca:

``` javascript

req.params = {
  id: '648ab123'
}

```

Entonces `getUser` funciona perfectamente.

### Pero para el apartado **"mi perfil"**

No buscamos que el usuario tenga que saber su ID.

Queremos algo más cómodo:

```

GET /api/v1/users/me

```

Aquí no existe un ID real.

Solo existe:

``` javascript

req.user

```

porque el middleware `protect` ya verificó el JWT y guardó el usuario autenticado:

``` javascript

req.user = currentUser;

```

Lo que hace getMe

Si el usuario autenticado es:

``` javascript

req.user = {
  id: '648ab123',
  name: 'Pepe'
}

```

entonces:

``` javascript

req.params.id = req.user.id;

```

convierte:

``` javascript

req.params = {
  id: '648ab123'
}

```

Ahora el siguiente middleware cree que recibió una ruta normal:

```

GET /users/648ab123

```

aunque realmente la petición fue:

```

GET /users/me

```
