# Params Middleware

## ¿Qué es `/:id` en una ruta?

Cuando hacemos algo así:

``` javascript

app.get('/users/:id', (req, res) => {
  res.send(req.params.id)
})

```

- `:id` es un parámetro dinámico en la URL.

Por ejemplo

```

GET /users/123

```

Entonces

``` javascript

req.params.id === "123"

```

Así que en la ruta

```

/users/

```

Esto no funcionaría

Si queremos que sea opcional, usamos `?`

Ejemplo

``` javascript

app.get('/users/:id?', (req, res) => {
  res.send(req.params.id || 'Sin ID')
})

```

Ahora sí funciona

```

/users
/users/123

```

# 1. ¿Qué es el param middleware?

Es un tipo especial de **middleware** que se **ejecuta cuando una ruta tiene cierto parámetro.**

Se define así:

``` javascript

app.param('id', (req, res, next, id) => {
  console.log('ID detectado:', id)
  next()
})

```

# 3. ¿Cuándo se ejecuta?

Cada vez que una ruta usa `:id`:

``` javascript

app.get('/users/:id', (req, res) => {
  res.send('Usuario ' + req.params.id)
})

```

Flujo:

- Llega request `/users/123`

- Se ejecuta `app.param('id', ...)`

- Luego entra al handler del endpoint

# 4. ¿Para qué sirve esto?

Sirve para lógica reutilizable sobre parámetros:

Ejemplo:

``` javascript

app.param('id', (req, res, next, id) => {
  // Buscar usuario en DB
  const user = users.find(u => u.id === id)

  if (!user) {
    return res.status(404).send('No existe')
  }

  req.user = user
  next()
})

app.get('/users/:id', (req, res) => {
  res.send(req.user)
})

```

Ventaja:

- No se repite la búsqueda en cada endpoint

- Se centraliza la lógica

# 5. Chaining Middleware

- Es poner varios middlewares uno tras otro en una misma ruta

Ejemplo

``` javascript

app.get('/ruta', middleware1, middleware2, middleware3, handler)

```

## 5.1 ¿Qué pasa aquí?

Cuando llega un request:

- Se ejecuta `middleware1`

- Luego `middleware2`

- Luego `middleware3`

- Finalmente el `handler`

Solo avanza si cada uno llama a:

``` javascript

next()

```

Ejemplo

``` javascript

const log = (req, res, next) => {
  console.log('Logging...')
  next()
}

const auth = (req, res, next) => {
  if (!req.headers.authorization) {
    return res.status(401).send('No autorizado')
  }
  next()
}

app.get('/perfil', log, auth, (req, res) => {
  res.send('Perfil del usuario')
})

```

Flujo:

Request → `/perfil`

- `log` → imprime → `next()`

- `auth` → valida → `next()`

- handler → responde

# Resumen

- `:id` → parámetro dinámico en la URL

- `:id?` → parámetro opcional

- `app.param()` → middleware que se ejecuta cuando ese parámetro aparece

- Sirve para validar, buscar datos, etc. antes del handler
