# 1. Thread Pool

En Node.js, el **Thread Pool** de libuv:

- por defecto tiene 4 threads

- puede configurarse

- el máximo es 128

Pero no se hace dentro del código JavaScript, sino mediante una **variable de entorno.**

# 2. Tamaño por defecto

El **thread pool** por defecto:

```
4 threads
```

Esto significa que solo 4 tareas pesadas pueden ejecutarse en paralelo.

Ejemplo de tareas que usan el **thread pool**:

- `fs.readFile`

- `fs.writeFile`

- `crypto.pbkdf2`

- `crypto.scrypt`

- `dns.lookup`

- `zlib`

# 3. Cómo aumentar el tamaño

Se usa la variable de entorno:

```
UV_THREADPOOL_SIZE
```

Ejemplo en Linux / Mac:

```
UV_THREADPOOL_SIZE=8 node app.js
```

Ejemplo en Windows (PowerShell):

```
$env:UV_THREADPOOL_SIZE=8
node app.js
```

# 4. Máximo permitido

El límite es:

```
128 threads
```

Ejemplo:

``` javascript
UV_THREADPOOL_SIZE=128 node app.js
```

# 5. Importante: no siempre conviene aumentarlo

Más `threads` no significa más rendimiento.

Si tenemos:
- CPU con 8 cores

- thread pool de 128

entonces el sistema hará mucho **context switching**, lo cual puede empeorar el rendimiento.

Regla general:

```
Thread pool ≈ número de cores o un poco más
```
### El thread pool NO se usa para todo el async de Node.

Por ejemplo:

- Esto NO usa thread pool:

- sockets

- HTTP requests

- TCP

- WebSockets

Porque eso lo maneja directamente el **Event Loop** de libuv usando `I/O` no bloqueante del sistema operativo.

El `thread pool` solo se usa cuando el sistema operativo **no tiene una API asincrónica.**

# 6. Flujo simplificado

```
JavaScript
   │
   ▼
V8 Engine
   │
   ▼
Event Loop
   │
   ├── I/O async (network) → OS
   │
   └── operaciones pesadas → Thread Pool (4–128 threads)
```

# 7. Flujo con Thread Pool

``` javascript
const crypto = require("crypto")

crypto.pbkdf2("password", "salt", 100000, 64, "sha512", () => {
  console.log("done")
})
```

Esto sí usa el `thread pool`.

Si hay 4 threads:

```
4 operaciones paralelas
```

Si ponemos:

```
UV_THREADPOOL_SIZE=16
```

podemos ejecutar 16 en paralelo.

# 8. Resumen

| Componente  | Función                |
| ----------- | ---------------------- |
| **V8**      | Ejecuta JavaScript     |
| **libuv**   | Maneja I/O asincrónico |
| Event Loop  | Coordina callbacks     |
| Thread Pool | Ejecuta tareas pesadas |
| Tamaño      | 4 por defecto          |
| Máximo      | 128                    |
