Node tiene 2 componentes importantes, que son el ***motor V8 de Google y Libuv***

# 1. V8 (motor de JavaScript)

V8 es el motor que creó **Google** para ejecutar JavaScript (también lo usa **Google Chrome**).

Su trabajo es:

- Interpretar y compilar JavaScript

- Convertir JS → código máquina

- Ejecutar funciones JS

- Manejar memoria (Garbage Collection)

Ejemplo:
```
let x = 5 + 10
```

**V8**:

- Lee el código

- Lo compila

- Lo ejecuta en la CPU

Pero **V8 NO sabe hacer cosas del sistema operativo como**:

- leer archivos

- hacer requests HTTP

- acceder a red

- manejar timers

Ahí entra el otro componente.

# 2. libuv

`libuv` es una librería escrita en `C` que le da a `**Node**` acceso al sistema operativo.

Se encarga de:

- I/O asincrónico

- filesystem

- networking

- timers

- thread pool

- event loop

Es básicamente la capa que habla con el sistema operativo.

# 3. El Event Loop

El `Event Loop` es parte de libuv.

Su trabajo es coordinar tareas asincrónicas.

`Node.js` funciona con un modelo llamado:

**Single Threaded Event Loop**

Esto significa:

- un solo thread principal

- que maneja muchas operaciones asincrónicas

## 3.1 Entonces ¿qué ejecuta el Event Loop?

El `Event Loop` ejecuta código **JavaScript rápido (no bloqueante)**.

Ejemplo:

``` javascript
console.log("hola")
console.log("mundo")
```

Esto lo ejecuta directamente **V8 en el thread principal**.

## 3.2 Cuando hay tareas pesadas

Cuando aparece algo como:

``` javascript
fs.readFile("file.txt", callback)
```

pasa esto:

- JS llama a **Node**
- **Node** delega a `libuv`
- `libuv` lo manda al `Thread Pool`

Luego cuando termina:

- el resultado vuelve al `Event Loop`
- el `Event Loop` ejecuta el `callback`

## 3.3. Thread Pool

### Thread Pool

El **thread pool** por defecto tiene **4 threads.**

Se usa para operaciones como:

- filesystem

- crypto

- DNS

- algunas operaciones de red

# Flujo
```
        JavaScript Code
               │
               ▼
             V8 Engine
               │
               ▼
           Event Loop
               │
       ┌───────┴────────┐
       ▼                ▼
   Callbacks         libuv
                         │
                         ▼
                    Thread Pool
                    (4 threads)
                         │
                         ▼
                     OS Kernel
    
```
## Con Código
``` javascript

const fs = require("fs")

console.log("start")

fs.readFile("file.txt", () => {
  console.log("file read")
})

console.log("end")
```

## Salida

```
start
end
file read
```

Porque:

- `start` → **event loop**

- `readFile` → **thread pool**

- `end` → **event loop**

- `callback` → cuando termina **thread pool**

# 4. Regla mental

En **Node.js**:

- **V8** → ejecuta JavaScript

- **libuv** → maneja I/O y async

- **Event Loop** → coordina callbacks

- **Thread Pool** → hace tareas pesadas

#  IMPORTANTE

Node.js is single-threaded but uses a thread pool for async I/O
