# Factory Functions

Es una función genérica que sirve para eliminar cualquier documento de cualquier modelo.

``` javascript

exports.deleteOne = Model =>
  catchAsync(async (req, res, next) => {
    const doc = await Model.findByIdAndDelete(req.params.id);

    if (!doc) {
      return next(new AppError('No document found with that ID', 404));
    }

    res.status(204).json({
      status: 'success',
      data: null
    });
  });

```

Recibe un parámetro:

``` javascript

Model =>

```

Ese `Model` puede ser cualquier modelo de Mongoose:

``` javascript

Tour
User
Review
Booking

```

## 1. Sin factory function

Tendríamos que escribir esto para `Tours`:

``` javascript

exports.deleteTour = async (req, res) => {
  await Tour.findByIdAndDelete(req.params.id);

  res.status(204).json({
    status: 'success',
    data: null
  });
};

```

Y después repetir casi exactamente lo mismo para `Users`:

``` javascript

exports.deleteUser = async (req, res) => {
  await User.findByIdAndDelete(req.params.id);

  res.status(204).json({
    status: 'success',
    data: null
  });
};

```

Y para `Reviews`:

``` javascript

exports.deleteReview = async (req, res) => {
  await Review.findByIdAndDelete(req.params.id);

  res.status(204).json({
    status: 'success',
    data: null
  });
};

```

Mucho código duplicado.

## 2. Con la factory function

La escribimos una sola vez:

``` javascript

exports.deleteOne = Model => ...

```

Y luego la reutilizamos.

Por ejemplo, en tourController.js:

``` javascript

const Tour = require('../models/tourModel');
const factory = require('./handlerFactory');

exports.deleteTour = factory.deleteOne(Tour);

```

Aquí ocurre:

``` javascript

factory.deleteOne(Tour)

```

Lo mismo que:

``` javascript

factory.deleteOne(Model)

```

donde:

``` javascript

Model = Tour

```

Entonces internamente queda:

``` javascript

Tour.findByIdAndDelete(req.params.id);

```

## 3. ¿Qué es exactamente `factory.deleteOne(Tour)`?

Es una función que devuelve otra función.

Ejemplo:

``` javascript

const saludar = nombre => {
  return () => {
    console.log(`Hola ${nombre}`);
  };
};

const saludoPepe = saludar('Pepe');

saludoPepe();

```

Salida:

```

Hola Pepe

```

Lo mismo pasa aquí:

``` javascript

const deleteTour = factory.deleteOne(Tour);

```

`deleteOne` recibe `Tour` y devuelve un middleware de Express:

``` javascript

async (req, res, next) => {
   ...
}

```

Por eso podemos hacer:

``` javascript

exports.deleteTour = factory.deleteOne(Tour);

```

porque el resultado final es una función válida para Express.

La idea es:

- `deleteOne` es una función genérica.

- Le pasas el modelo (`Tour`).

- Devuelve un middleware de Express.

- Ese middleware elimina el documento correspondiente.

Este patrón se conoce como **Higher-Order Function** (función que recibe o devuelve funciones) y es una de las razones por las que el código de controladores puede mantenerse muy limpio en proyectos grandes con Express y Mongoose.
