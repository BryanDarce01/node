# CACHING

Después de ejecutar un módulo, Node lo **guarda en memoria (cache)**

## 1. Qué significa esto?

La primera vez que hacemos:

``` javascript

require('./miModulo');

```

Node:

- Lo busca

- Lo envuelve

- Lo ejecuta

Guarda el resultado (`module.exports`) en cache

La segunda vez

``` javascript

require('./miModulo');

```

Node:

- NO lo vuelve a ejecutar

- Simplemente devuelve lo que ya guardó

## 2. ¿Dónde guarda eso Node?

Internamente en algo como:

``` javascript

require.cache

```

- Es un objeto donde **Node** guarda todos los módulos ya cargados

## 3. ¿Por qué existe el caching?

### 1. Performance

- No vuelve a leer archivos

- No vuelve a ejecutar código

## 4. ¿Se puede limpiar el cache?

Sí (pero no es común):

``` javascript

delete require.cache[require.resolve('./miModulo')];

```

- Esto fuerza a **Node** a volver a cargar el módulo

## 5. Conexión final de TODO el flujo

Cuando hacemos:

``` javascript

require('algo');

```

Node hace:

- **Resolving** → encuentra el módulo

- **Wrapping** → lo mete en una función

- **Execution** → ejecuta el código

- **Returning exports** → devuelve `module.exports`

- **Caching** → guarda el resultado

## 6. Idea final

- Un módulo en **Node** es como un s**ingleton por defecto**

- Se ejecuta una vez

- Se comparte en toda la app
