# RETURNING EXPORTS

Aquí es donde `require()` finalmente devuelve algo

![returning_exports](../../img/requiring_modules/03.%20returning_exports.png)

## La idea clave

- `require()` SIEMPRE devuelve lo que esté en:

``` javascript

module.exports

```

## Ejemplo

- **archivo:** `math.js`

``` javascript

module.exports = {
  suma: (a, b) => a + b
};

```
- **archivo:** `app.js`

``` javascript

const math = require('./math');

console.log(math.suma(2, 3));

```

Aquí `require('./math')` devuelve:

``` javascript

{ suma: (a, b) => a + b }

```

## Entonces… ¿qué pasó internamente?

Durante **EXECUTION**:

``` javascript

module.exports = { suma };

```

Y ahora en **RETURNING EXPORTS**:

``` javascript

return module.exports;

```

## Diferencia entre `module.exports` vs `exports`

### 1. `module.exports` (EL IMPORTANTE)

- Es el objeto real que se devuelve

``` javascript

module.exports = function () {
  console.log("Hola");
};

```

### 2. `exports` (solo un atajo)

``` javascript

exports.suma = (a, b) => a + b;

```

- Internamente es como:

``` javascript

module.exports.suma = ...

```

### 3. Error

``` javascript

exports = function () {};

```

- Esto rompe todo

Porque:

- `exports` deja de apuntar a `module.exports`

- `require()` devuelve `{}` vacío

### 4. IMPORTANTE

- Si queremos exportar una sola cosa:

``` javascript

module.exports = algo;

```

- Si queremos exportar varias cosas:

``` javascript

exports.a = 1;
exports.b = 2;

```

### 5. Ejemplo

- Caso 1: un solo `export`

``` javascript

module.exports = "Hola mundo";

```

``` javascript

const msg = require('./file');
console.log(msg); // "Hola mundo"

```

- Caso 2: múltiples exports

``` javascript

exports.a = 1;
exports.b = 2;

```

``` javascript

const obj = require('./file');
console.log(obj); // { a: 1, b: 2 }

```

### 6. Conexión

EXECUTION → define `module.exports`

RETURNING EXPORTS → **Node** hace:

``` javascript

return module.exports;

```

### 7. Resumen

- `require()` no devuelve el archivo

- No devuelve funciones mágicas

- Devuelve EXACTAMENTE lo que pongamos en:

``` javascript

module.exports

```
