# WRAPPING

![wrapping](../../img/requirin_modules/02.%20wrapping.png)

Cuando Node ya encontró el módulo (paso 1), no ejecuta el archivo directamente.

- Primero lo **envuelve (wrap)** dentro de una función.

## ¿Cómo lo envuelve?

Internamente **Node** hace algo como esto:

``` javascript

(function (exports, require, module, __filename, __dirname) {
  // Código aquí
});

```

- Esto NO lo escribimos nosotros, **Node** lo hace automáticamente.

## Esto se ejecuta en Run Time

Significa que este “envoltorio” se crea justo cuando ejecutamos el programa, no antes.

O sea:

- No existe físicamente en nuestro archivo

- Node lo genera dinámicamente

- Luego ejecuta esa función

## ¿Para qué sirve este wrapping?

**Node** nos da estas 5 “variables mágicas” dentro de cada archivo:

### 1. `require`

- Para importar otros módulos

``` javascript

const fs = require('fs');

```

### 2. `module`

- Representa el módulo actual

``` javascript

module.exports = {};

```

### 3. `exports`

- Atajo de module.exports

``` javascript

exports.saludo = "hola";

```
### 4. `__filename`

- Ruta completa del archivo actual

``` javascript

console.log(__filename);

```

### 5. `__dirname`

- Carpeta donde está el archivo

``` javascript

console.log(__dirname);

```

## Resumen

Gracias al wrapping:

- Cada archivo en Node tiene su propio scope (aislado)

- No contamina variables globales

- Puedemos usar `require`, `exports`, etc. sin declararlos
