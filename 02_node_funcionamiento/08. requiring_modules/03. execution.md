# EXECUTION

Después de:

- Encontrar el módulo [Resolving](01.%20requiring_modules.md)

- Envolverlo en una función [Wrapping](02.%20wrapping.md)

Ahora el runtime de **Node.js** ejecuta esa función.

## 1. ¿Qué significa “ejecutar”?

Básicamente esto:

``` javascript

(function (exports, require, module, __filename, __dirname) {
  // Nuestro código aquí
})();

```

- Node llama esa función inmediatamente.

## 2. ¿Qué pasa durante esa ejecución?

Aquí es donde realmente sucede todo:

### 1. Se corre el código

``` javascript

console.log("Hola");

```

### 2. Se crean variables

``` javascript

const x = 10

```

### 3. Se ejecutan `requires` internos

``` javascript

const fs = require('fs');

```

### 4. Se definen `exports`

``` javascript

module.exports = { suma };

```

## IMPORTANTE

El código solo se ejecuta 1 vez.

Aunque hagamos:

``` javascript

require('./miModulo');
require('./miModulo');

```

No se ejecuta dos veces
- Porque después entra el paso 5 (caching)

## Resumen

- “Node ejecuta el código del módulo y listo”

Eso significa:

- Se ejecuta dentro de una función

- Puede disparar otros requires

- Define lo que exporta

- Solo ocurre una vez
