# Requiring Modules

![requiring_modules](../../img/requirin_modules/01.%20resolving_and_loading.png)

## 1. Start with core modules

Cuando usamos `require()` en **Node.js**, lo primero que hace internamente es revisar si estamos pidiendo un módulo nativo **(core module)**.

## 2. ¿Qué son los core modules?

Son módulos que ya vienen **incluidos con Node.js**, no necesitamos instalarlos.

Ejemplos:

```
require('http');
require('fs');
require('path');
```

## 3. ¿Qué pasa en este paso 1?

Cuando escribimos:

```
require('http');
```

**Node** hace esto:

“¿`http` es un módulo nativo mío?”

- Si **sí lo es** → lo carga inmediatamente

- Si **no lo es** → pasa al siguiente paso (Wrapping)


## 4. ¿Por qué este paso va primero?

Porque es lo más rápido y seguro:

- No necesita buscar archivos en nuestro proyecto

- No necesita ir a `node_modules`

- Ya está precompilado dentro de Node

## Ejemplo

``` javascript
const http = require('http');
```

Aquí:

- **Node** reconoce `http` como core module

- Lo carga directamente

- **No busca en carpetas ni archivos**

## 5. Idea clave (muy importante)

- Si NO usamos `./` o `../`, **Node asume primero que es un core module**

Por ejemplo:

| Código               | Qué intenta primero        |
| -------------------- | -------------------------- |
| `require('http')`    | Core module                |
| `require('express')` | Core → luego npm           |
| `require('./file')`  | Archivo local (salta core) |
