# Semantic Versioning o SemVer (versionado semántico)

Es el sistema que usan `npm` y la mayoría de paquetes de `Node.js`para manejar versiones.

#  Las versiones siguen este formato:
```
MAJOR.MINOR.PATCH
```

Ejemplo:

```
3.0.0
```

# 1. Qué significa cada número

## 1. MAJOR
```
3.0.0
↑
```
- Cambios grandes que rompen compatibilidad (breaking changes).

Ejemplo:

``` javascript
// versión 2
app.listen(port)

// versión 3
server.start(port)
```
Si el código dependía de la versión 2, podría dejar de funcionar en la 3.

Por eso `npm` no actualiza automáticamente `major versions` cuando usa `^`.

## 2. MINOR

```
3.1.0
  ↑
```

- nuevas funcionalidades

- no rompe compatibilidad

Ejemplo:

``` javascript
axios.get()
axios.post()
axios.put() // nuevo método agregado

```

El código sigue funcionando igual.

## 3. PATCH

```
3.1.2
    ↑
```

- corrección de bugs

- mejoras pequeñas

- sin cambiar comportamiento

Ejemplo:

- arreglar memory leak

- arreglar error interno

# 2. Qué significa `^`

El símbolo `^` (caret) significa:

- Permite actualizar `minor` y `patch`, pero no `major`.

Entonces `npm` acepta:

```
3.0.1
3.0.5
3.1.0
3.2.4
```
Pero no:

```
4.0.0
```

Esto evita romper la aplicación.

## Ejemplo con Nodemon:

```
^3.0.0
```

permitido:

- 3.X.X

# 3. Qué significa `~`

El símbolo `~` (tilde) significa:

- Solo permite actualizar `patch`.

Permitido:

```
3.0.1
3.0.2
3.0.9
```

No permitido:

```
3.1.0
4.0.0
```

# 4. Qué significa `*`

``` javascript
"nodemon": "*"
```

Significa:

- cualquier versión

- `npm` instalará la más reciente disponible.

**Esto es muy peligroso en producción.**

# 5. Comparación

| Símbolo  | Qué permite       |
| -------- | ----------------- |
| `*`      | cualquier versión |
| `^3.0.0` | minor + patch     |
| `~3.0.0` | solo patch        |
| `3.0.0`  | versión exacta    |
