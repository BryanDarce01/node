# REST

Representational State Transfer

En español:

**“Transferencia del estado representacional”**

- REST es una forma de diseñar APIs
- No es una tecnología
- No es un lenguaje
- Es un conjunto de reglas / estilo

REST dice:

- **“Accede a datos (recursos) usando URLs y métodos HTTP”**

## 1. Representational

Lo que recibe el programa es una representación del dato

Ejemplo:

```

{
  "id": 1,
  "name": "Bryan"
}

```

- No es el objeto real, es una **representación (JSON)**

## 2. State (estado)

Es el estado de algo en el servidor

Ejemplo:

```

Usuario activo/inactivo
Pedido enviado/no enviado
Producto disponible/no disponible

```

## 3. Transfer

Ese estado se transfiere entre cliente y servidor

Ejemplo:

- El programa hace GET → recibe datos

- El programa hace POST → cambias datos

## Ejemplo

1. Petición:

```
GET /users/1
```

2. Respuesta:

``` json
{
  "id": 1,
  "name": "Bryan"
}
```

Se está transfiriendo el estado del usuario en forma de **JSON**

# Resumen

Es seguir estas reglas:

 1. Usar recursos (nombres)
 ```

/users
/products
/orders

```

 2. Usar métodos HTTP

```

    GET → obtener
    POST → crear
    PATCH → actualizar
    DELETE → eliminar

```
 3. Ser “stateless” (sin estado)

- Cada request es independiente

Ejemplo:

- No guarda memoria entre peticiones

- Cada request trae toda la info necesaria (token, datos, etc.)

## Ejemplo con Express

``` javascript

app.get('/users', ...)        // obtener usuarios
app.post('/users', ...)       // crear usuario
app.patch('/users/:id', ...)  // actualizar
app.delete('/users/:id', ...) // eliminar

```

# Resumen corto

**REST = estilo para construir APIs**

Usa:

- URLs limpias (recursos)
- Métodos HTTP
- JSON como representación
- Requests independientes (stateless)
