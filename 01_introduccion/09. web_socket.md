# WebSocket 
Es un protocolo que permite una **comunicación bidireccional y en tiempo real** entre un cliente (por ejemplo, el navegador) y un servidor.

## Con WebSockets:

- Cliente y servidor pueden **enviarse datos en cualquier momento**
- La conexión se mantiene **abierta** (no se cierra después de cada request)

## Comparación con HTTP

Con HTTP (lo normal)

Funciona como **request → response**

El cliente pide, el servidor responde, y se termina

```
Cliente → request → Servidor
Cliente ← response ← Servidor
(conexión cerrada)
```

## WebSocket

- Conexión **persistente**

- Comunicación en **ambos sentidos**

```
Cliente ⇄ Servidor
(conexión siempre abierta)
```

## Casos donde se usa WebSocket:

- Chats (WhatsApp Web)

- Juegos online

- Notificaciones en tiempo real

- Trading / precios en vivo

## Cómo funciona técnicamente

- Empieza como una petición HTTP normal

- Se hace un "upgrade" a WebSocket

- Se mantiene la conexión abierta

## WebSocket en Node.js

En Node, los WebSockets se manejan normalmente con librerías como:

- `ws`

- `socket.io`

## Ejemplo

**Servidor**
``` javascript

const WebSocket = require("ws");

const server = new WebSocket.Server({ port: 3000 });

server.on("connection", socket => {
  console.log("Cliente conectado");

  socket.on("message", message => {
    console.log("Mensaje:", message.toString());
  });

  socket.send("Hola cliente!");
});

```

**Cliente**

``` javascript

const socket = new WebSocket("ws://localhost:3000");

socket.onmessage = event => {
  console.log(event.data);
};

socket.send("Hola servidor!");

```
## Un WebSocket en Node es como un Duplex Stream

Porque:

- Puede `leer datos` (incoming messages)

- Puede `escribir datos` (send messages)

Al mismo tiempo.

## Resumen

```
HTTP = pedir y responder (una vez)
WebSocket = conversación continua
```


## Importante

WebSockets:

- Son más eficientes para tiempo real

- Pero más complejos que HTTP

- Mantienen conexiones abiertas (más consumo de recursos)
