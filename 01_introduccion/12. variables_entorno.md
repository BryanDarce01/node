# Variables de Entorno

Son variables que viven fuera del código, en el sistema.

Node las lee con:

``` javascript

process.env

```

## Archivo `.env`

Es un archivo donde guardamos variables de entorno (configuración de al app).

- Son valores sensibles o configurables que NO queremos poner directo en el código

## ¿Qué tipo de cosas se guardan ahí?

Ejemplo típico:

```

PORT=3000
DB_URI=mongodb://localhost:27017/miapp
API_KEY=123456
JWT_SECRET=mi_secreto_super_seguro

```

## ¿Por qué usar `.env`?

❌ MAL (hardcodeado en el código)

``` javascript

const password = "123456"

```

Problemas:

- inseguro
- si se suve a Git, exponemos datos
- difícil cambiar en producción

✅ BIEN (con `.env`)

```

const password = process.env.DB_PASSWORD

```

Ventajas:

- más seguro

- fácil cambiar según entorno

- código limpio

## Usar `.env` en Node

Se usa normalmente la librería:

- dotenv

La instalamos

``` bash

npm install dotenv

```

Creamos el archivo `.env`

```

PORT=3000
DB_URI=mongodb://localhost:27017/test

```

Cargamos las variables

```

require('dotenv').config()

```

La usamos

``` javascript

const port = process.env.PORT

app.listen(port, () => {
  console.log(`Servidor en puerto ${port}`)
})

```

## NUNCA SUBIR EL `.env` A GIT

## Diferentes entornos

Podemos tener distintos `.env`:

- `.env.development`

- `.env.production`

- `.env.test`

Cambian dependiendo dónde corre la app.

## Resumen

- `.env` = archivo de configuración

- guarda datos sensibles o variables

- se usa con `process.env`

- se carga con `dotenv`

- ❌ nunca subirlo a Git
