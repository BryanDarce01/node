# Arquitectura `client–server` o modelo `request–response`

Es una forma de diseñar sistemas donde un programa (cliente) pide algo y otro programa (servidor) responde. Es uno de los modelos más usados en Internet.

# 1. Idea clave

Hay dos roles:

**Client (cliente)**
Es el programa que hace la solicitud (`request`).

**Server (servidor)**
Es el programa que recibe la solicitud y envía una respuesta (`response`).

El flujo básico es:

```

Client  ── request ──► Server
Client  ◄─ response ── Server

```

# 2. Características del modelo `Request–Response`

- **El cliente inicia la comunicación**
    - El servidor no responde si nadie pide algo.

- Cada `request` tiene una response

```

1 request → 1 response

```

- **Es stateless en HTTP (normalmente)**
    - Cada request es independiente.

# 3. Diferencias

Aunque se usan como sinónimos, técnicamente:

**Client-Server Architecture**

- Es el modelo de sistema (quién es cliente y quién servidor).

**Request-Response Model**

- Es el patrón de comunicación (petición → respuesta).
