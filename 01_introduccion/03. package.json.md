# Node Package Manager.

Es el gestor de paquetes de Node.js.

Su función principal es instalar, administrar y compartir librerías (paquetes) de JavaScript.

En otras palabras:

- npm es la herramienta que permite instalar código de otros desarrolladores y usarlo en tu proyecto.

# 1. Qué es un "paquete"

Un paquete (package) es una librería o módulo reutilizable.

Ejemplos famosos:

- Express.js → framework para crear servidores

- Lodash → utilidades para trabajar con arrays y objetos

- Axios → hacer requests HTTP

Con npm se puede instalarlos así:

```

npm install express

```

`npm` hace varias cosas automáticamente:

- descarga el paquete desde el registro de `npm`

- lo guarda en la carpeta `node_modules`

- lo agrega al archivo `package.json`

# 2. Qué hacemos con `npm init`

Cuando se ejecuta:

```

npm init

```

`npm` crea un archivo llamado:

- `package.json`

Este archivo describe el proyecto.

Ejemplo:

``` javascript

{
  "name": "mi-proyecto-nuevo",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  }
}

```

Ahí se guardan:

- nombre del proyecto

- versión

- dependencias

- scripts

- configuración

# 3. Cómo funciona scripts

Dentro de "scripts" definimos comandos personalizados.

Ejemplo:

```
"scripts": {
  "start": "node index.js",
  "dev": "nodemon index.js",
  "test": "jest"
}
```
Luego ejecutamos:

```
npm start
npm run dev
npm test
```
## 3.1 Regla importante de `npm`

Hay scripts que no necesitan run:

```
npm start
npm test
npm stop
```
Pero los demás sí:

```
npm run dev
npm run build
```

# 4.  `npm` sirve para:

- Instalar paquetes

```
npm install express
```

- Eliminar paquetes

```
npm uninstall express
```

- Ejecutar scripts

```
npm run start
```

- Actualizar paquetes

```
npm update
```

# Formas de instalación

## 1. Instalación normal (dependencias de producción)

```
npm install express
```

Esto agrega el paquete en package.json dentro de:

``` javascript
"dependencies": {
  "express": "^4.18.2"
}
```

Significa que la aplicación necesita ese paquete para funcionar en producción.

### Ejemplos típicos:

- Express.js

- Axios

- Lodash

Son librerías que el código usa cuando la app está corriendo.


## 2. Instalación de desarrollo `(--save-dev)`
```
npm install nodemon --save-dev
```

o forma corta:

```
npm install nodemon -D
```

Esto se guarda en package.json en:

``` javascript
"devDependencies": {
  "nodemon": "^3.0.0"
}
```

Estas dependencias solo se usan para desarrollar, no para ejecutar la app en producción.

Ejemplos:

- Nodemon → reinicia el servidor automáticamente

- ESLint → revisar errores de código

- Prettier → formatear código

- Jest → testing

## Diferencias

| Tipo            | Se usa cuando         | Ejemplo |
| --------------- | --------------------- | ------- |
| dependencies    | la app está corriendo | express |
| devDependencies | desarrollo            | nodemon |


# Resumen

| Herramienta  | Qué es                           |
| ------------ | -------------------------------- |
| Node.js      | runtime para ejecutar JavaScript |
| npm          | gestor de paquetes de Node       |
| package.json | archivo que describe tu proyecto |
