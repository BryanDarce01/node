# 1. DNS

El **DNS (Domain Name System)** es el sistema que traduce nombres de dominio que los humanos entendemos en **direcciones IP que las computadoras** usan para comunicarse en Internet. 

# 1. La idea básica

Los humanos usamos nombres como:

```

Google → google.com

```

Pero las computadoras realmente se comunican con direcciones IP como:

```
142.250.190.78
```

## Algo interesante (muy usado en backend)

El **DNS** no solo traduce nombres, también se puede indicar:

```
servidores de correo

balanceo de carga

subdominios

CDN
```

Por ejemplo:

```
api.midominio.com

cdn.midominio.com

mail.midominio.com
```

Todo eso se configura en DNS.

# 2. ¿Qué es una URL?

Una **URL (Uniform Resource Locator)** es la dirección completa que se usa para acceder a un recurso en internet.

Ejemplo:

```
https://api.midominio.com:3000/users?id=10#perfil
```

## 2.1 Partes de una URL

```
https://api.midominio.com:3000/users?id=10#perfil
```

### 2.1.1 Protocolo
```
https://
```

Indica **cómo se comunica el cliente con el servidor**

Ejemplos:

- `http`
- `https` (seguro 🔒)
- `ftp`

Aquí entra el tema de redes como [TCP/IP](08.%20tcp-ip.md)

### 2.1.2 Subdominio

```
api
```

- Es una **división del dominio principal**

- Se usa mucho en backend para separar servicios

Ejemplos:

```
api.midominio.com
cdn.midominio.com
mail.midominio.com
```

Esto se configura en DNS

### 2.1.3 Dominio

```
midominio
```

- Es el nombre principal del sitio

- Es lo que se compra o registra

### 2.1.4 TLD (Top-Level Domain)

```
.com
```

- Es la extensión del dominio

Ejemplos:

- `.com`
- `.org`
- `.net`
- `.gt` (Guatemala 🇬🇹)

## Todo esto junto:

```
api.midominio.com
```

Se llama:

- **Hostname (o dominio completo)**

- Esto es lo que el **DNS traduce a una IP**

### 2.1.5 Puerto (opcional)

```
:3000
```

- Es el canal específico dentro del servidor

Ejemplos:

- `80-` → HTTP

- `443-` → HTTPS

- `3000-` → común en Node.js

Si no lo ponemos, el navegador usa uno por defecto

### 2.1.6 Path (ruta)

```
/users
```

- Indica qué recurso queremos dentro del servidor

Ejemplos:

```
/users
/products
/api/login
```

- En Express esto serían nuestras rutas.

### 2.1.7 7. Query params (parámetros)

```
?id=10
```

- Datos que se le manda al servidor

Ejemplo

```
?id=10&activo=true
```

- En Node/Express los leemos así:

``` javascript
req.query.id
```

### 2.1.8 Fragment (hash)

```
#perfil
```

- Se usa en el frontend
- No se envía al servidor

Ejemplo:

- Scroll a una sección
- Navegación en SPA (React, etc.)

## 🧠 Conexión IMPORTANTE (DNS + URL)

Cuando escribimos:

```
https://api.midominio.com/users
```

Pasa esto:

1. El navegador toma:

```
api.midominio.com
```

2. Va al DNS y dice:

    - "¿Cuál es la IP de esto?"

3. El DNS responde:

```
142.250.190.78
```

4. Luego se hace la petición al servidor real


## Resumen rápido

Una URL es:

```
protocolo://subdominio.dominio.tld:puerto/path?query#fragment
```
