# Virtual Populate 

Es una funcionalidad de Mongoose.

- Sirve para crear una relación entre colecciones sin guardar físicamente la referencia en ambos documentos.

### Ejemplo

#### Usuarios

``` javascript

{
  _id: "u1",
  name: "Bryan"
}

```

#### Tours

``` javascript

{
  _id: "t1",
  name: "Tour a Tikal",
  guides: ["u1"]
}

```

- Cada **tour** guarda los IDs de los usuarios que son guías.

Ahora si queremos obtener:

- "Un usuario junto con todos los tours donde es guía"

Pero el documento `Usuarios` no tiene una propiedad `tours`.

Podríamos agregar un arreglo `tours` en `Usuarios`, pero estaríamos duplicando información.

## Solución: Virtual Populate

Creamos un campo virtual:

``` javascript

userSchema.virtual('tours', {
  ref: 'Tour',
  foreignField: 'guides',
  localField: '_id'
});

```

Esto significa:

- `ref`: modelo que quiero buscar

- `foreignField`: campo en Tour que contiene la referencia

- `localField`: campo actual del User que usaré para buscar

Significa:

- Buscar todos los Tours cuyo campo `guides` contenga el `_id` de este usuario y colócalos en la propiedad virtual `tours`.

### Luego hacemos populate

``` javascript

const user = await User.findById(id).populate('tours');

```

Resultado:

``` javascript

{
  _id: "u1",
  name: "Bryan",
  tours: [
    {
      _id: "t1",
      name: "Tour a Tikal"
    },
    {
      _id: "t2",
      name: "Tour a Antigua"
    }
  ]
}

```

**Importante**:

- `tours` no existe en MongoDB.

- Se genera dinámicamente cuando haces `populate()`.

Por eso se llama **virtual**.

## Diferencia entre populate normal y virtual populate

### 1. Populate normal

El documento tiene la referencia guardada.

``` javascript

{
  name: "Tour a Tikal",
  guides: [
    ObjectId("u1")
  ]
}

```

``` javascript

Tour.find().populate('guides');

```

### 2. Virtual Populate

La referencia no existe en el documento actual.

``` javascript

{
  _id: "u1",
  name: "Bryan"
}

```

Creamos una relación virtual:

``` javascript

userSchema.virtual('tours', ...)

```

y luego:

``` javascript

User.find().populate('tours');

```

Mongoose busca los tours relacionados aunque el usuario no tenga almacenado ningún campo `tours`.

# Resumen

Usamos **populate normal** cuando el documento ya tiene un `ObjectId` que apunta a otro documento:

``` javascript

review.user
review.tour
tour.guides

```

Usamo **virtual populate** cuando necesitamos navegar la relación en sentido contrario sin duplicar datos.

```

Tour  -----> Reviews

```

Cada review guarda:

``` javascript

{
  review: "Excelente",
  tour: ObjectId("tour1")
}

```

Pero el Tour no guarda:

``` javascript

reviews: [...]

```
Entonces se crea:

``` javascript

tourSchema.virtual('reviews', {
  ref: 'Review',
  foreignField: 'tour',
  localField: '_id'
});

```

y después:

``` javascript

Tour.findById(id).populate('reviews');

```

- para obtener todas las reviews del tour **sin almacenar un arreglo de reviews** dentro del documento `Tour`.
