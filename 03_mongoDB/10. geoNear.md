# `geoNear`

`$geoWithin` responde:

- ¿Qué documentos están dentro de esta área?

`$geoNear` responde:

- ¿Qué tan lejos está cada documento de este punto?

y además los devuelve ordenados por distancia.

## Ejemplo

Nuestra app tiene estos tours

``` javascript

[
  {
    name: "Tour A",
    startLocation: {
      type: "Point",
      coordinates: [-90.50, 14.60]
    }
  },
  {
    name: "Tour B",
    startLocation: {
      type: "Point",
      coordinates: [-90.55, 14.55]
    }
  }
]

```

Consulta:

``` javascript

const distances = await Tour.aggregate([
  {
    $geoNear: {
      near: {
        type: 'Point',
        coordinates: [-90.734, 14.558]
      },
      distanceField: 'distance'
    }
  }
]);

```

Y el usuario está en:

- Longitud: -90.52

- Latitud: 14.58

Con `$geoNear`, MongoDB devuelve:

``` javascript

[
  {
    name: "Tour B",
    distance: 3.2
  },
  {
    name: "Tour A",
    distance: 8.7
  }
]

```

`geoNear`:

- Calcula las distancias.

- Ordena automáticamente del más cercano al más lejano.

## Requisitos

### 1. Campo GeoJSON

``` javascript

startLocation: {
  type: {
    type: String,
    default: 'Point',
    enum: ['Point']
  },
  coordinates: [Number]
}

```

### 2. Índice geoespacial

``` javascript

tourSchema.index({ startLocation: '2dsphere' });

```

- Sin este índice, `$geoNear` no funcionará.

### 3. Se usa dentro de `aggregate()`

``` javascript

Tour.aggregate([
  {
    $geoNear: {
      ...
    }
  }
]);

```

- Porque `$geoNear` es una etapa (stage) del Aggregation Pipeline.

### 4. `distanceMultiplier`

Podemos convertir metros a kilómetros:

``` javascript

const multiplier = 0.001;

```

y hacer:

``` javascript

const distances = await Tour.aggregate([
  {
    $geoNear: {
      near: {
        type: 'Point',
        coordinates: [-90.734, 14.558]
      },
      distanceField: 'distance',
      distanceMultiplier: multiplier
    }
  }
]);

```

Resultado:

``` javascript

[
  {
    name: "The Forest Hiker",
    distance: 1.45
  },
  {
    name: "The Sea Explorer",
    distance: 8.2
  }
]

```

- Ahora las distancias están en kilómetros.

## Ejemplo de un controller

Ruta:

``` javascript

router.get(
  '/distances/:latlng/unit/:unit',
  tourController.getDistances
);

```

Petición:

```

/distances/14.558,-90.734/unit/km

```

Controller:

``` javascript

exports.getDistances = async (req, res) => {
  const { latlng, unit } = req.params;

  const [lat, lng] = latlng.split(',');

  const multiplier =
    unit === 'mi'
      ? 0.000621371
      : 0.001;

  const distances = await Tour.aggregate([
    {
      $geoNear: {
        near: {
          type: 'Point',
          coordinates: [lng * 1, lat * 1]
        },
        distanceField: 'distance',
        distanceMultiplier: multiplier
      }
    },
    {
      $project: {
        distance: 1,
        name: 1
      }
    }
  ]);

  res.status(200).json({
    status: 'success',
    data: {
      data: distances
    }
  });
};

```

## ¿Por qué usar `$project`?

Porque después de calcular las distancias talvez solo queremos devolver:

``` javascript

{
  distance: 4.7,
  name: "The Forest Hiker"
}

```

- en vez de devolver todo el documento.

## Importante

Dentro del pipeline de agregación:

``` javascript

Tour.aggregate([
  {
    $geoNear: { ... }
  },
  {
    $project: { ... }
  }
]);

```

- `$geoNear` debe ser siempre la primera etapa del pipeline.

## Resumen

| Operador                               | Función                                   |
| -------------------------------------- | ----------------------------------------- |
| `$geoWithin`                           | Encontrar documentos dentro de una región |
| `$geoNear`                             | Calcular distancias desde un punto        |
| Requiere índice `2dsphere`             | Sí                                        |
| Devuelve distancia                     | Solo `$geoNear`                           |
| Usa `aggregate()`                      | Sí                                        |
| Debe ser la primera etapa del pipeline | Sí                                        |
| Distancia por defecto                  | Metros                                    |
| Puede convertirse a km o millas        | Con `distanceMultiplier`                  |

En otras palabras:

- `$geoWithin` → ¿Quiénes están dentro?

- `$geoNear` → ¿Qué tan lejos están?
