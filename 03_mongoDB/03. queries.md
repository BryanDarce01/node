# Hacer queries en mongoDB

## 1. `.find()` — lo básico

``` javascript

db.usuarios.find()

```

- Trae todos los documentos de la colección

## 2. Con filtro

``` javascript

db.usuarios.find({ nombre: "Bryan" })

```

- Trae solo los que cumplan esa condición

## 3. Operadores (`$gt`, `$lt`, etc.)

Estos son como los operadores en SQL (`>`, `<`, etc.)

### 3.1 `$gt` (greater than = mayor que)

``` javascript

db.usuarios.find({ edad: { $gt: 18 } })

```

- Usuarios con edad mayor a 18

### 3.2 `$lt` (less than = menor que)

``` javascript

db.usuarios.find({ edad: { $lt: 30 } })

```

- Menores de 30

## 4. Combinados

``` javascript

db.usuarios.find({
  edad: { $gt: 18, $lt: 30 }
})

```

- Entre 18 y 30

## 5. `$or`

Se le debe de pasar un `array` al operador `$or` por eso se usan los `[ ]` y adentro van los objetos

``` javascript

db.usuarios.find({
  $or: [
    { nombre: "Bryan" },
    { edad: { $lt: 20 } }
  ]
})

```

Trae:

- los que se llaman Bryan
O los menores de 20

## 6. `{ name: 1 }` → PROYECCIÓN

Esto es importante.

- Sirve para decir qué campos queremos que devuelva.

Ejemplo

``` javascript

db.usuarios.find(
  { edad: { $gt: 18 } },
  { nombre: 1 }
)

```

## 7. Ocultar `_id`

``` javascript

db.usuarios.find(
  {},
  { nombre: 1, _id: 0 }
)

```

# Con SQL

``` sql

SELECT nombre FROM usuarios WHERE edad > 18;

```

# Con mongoDB

``` javascript

db.usuarios.find(
  { edad: { $gt: 18 } },
  { nombre: 1 }
)

```

## Ejemplo real

``` javascript

db.usuarios.find(
  {
    $or: [
      { edad: { $gt: 25 } },
      { nombre: "Ana" }
    ]
  },
  {
    nombre: 1,
    edad: 1,
    _id: 0
  }
)

```

Significa:

- Usuarios con edad > 25 O nombre Ana

- Solo devuelve nombre y edad

## Resumen

- `.find()` → buscar datos

- `{}` → condiciones

- `$gt`, `$lt` → comparaciones

- `$or` → múltiples condiciones

- `{ campo: 1 }` → qué mostrar

# Para actualizar

## 1. `updateOne`

``` javascript

db.usuarios.updateOne(
  { nombre: "Bryan" },   // filtro
  { $set: { edad: 26 } } // cambio
)

```

- Solo actualiza un documento (el primero que encuentre)

## 2. updateMany

``` javascript

db.usuarios.updateMany(
  { edad: { $lt: 18 } },
  { $set: { menor: true } }
)

```

- Actualiza todos los que cumplan la condición

## IMPORTANTE

### `$set` — clave para crear o actualizar campos

``` javascript

{ $set: { nuevoCampo: valor } }

```
- Si el campo existe → lo actualiza

- Si no existe → lo crea automáticamente

Ejemplo

``` javascript

db.usuarios.updateMany(
  { edad: { $gt: 18 } },
  { $set: { esAdulto: true } }
)

```

## Esto SOLO funciona así:

``` javascript

[
  { $set: { ... } }
]

```

- (array) = aggregation pipeline

- sin array = update normal (no puedes usar `$concat`, `$cond`, etc.)

## Resumen

- `updateOne` → uno

- `updateMany` → varios

- `$set` → crea o actualiza campos

- `[]` → modo avanzado (pipeline)

- `$concat`, `$multiply`, `$cond` → usar valores de otros campos

# Para borrar

## 1. `deleteOne`

``` javascript

db.usuarios.deleteOne({ nombre: "Bryan" })

```
- Borra solo el primer documento que coincida

## 2. `deleteMany`

``` javascript

db.usuarios.deleteMany({ edad: { $lt: 18 } })

```

- Borra todos los documentos que cumplan la condición

## 3. El caso peligroso: `{}`

``` javascript

db.usuarios.deleteMany({})

```

Esto significa:

- **“borra TODO”**

Porque:

- `{}` = sin filtro

- ⇒ todos los documentos coinciden

### Resultado

- La colección queda vacía

- Pero la colección sigue existiendo

## 4. Diferencia con `drop()`

### 4.1 `deleteMany({})`

``` javascript

db.usuarios.deleteMany({})

```

- Borra los documentos

- ❌ NO borra la colección

### 4.2 `drop()`

``` javascript

db.usuarios.drop()

```

Este borra:

- documentos

- estructura

- índices

❌ elimina completamente la colección

### Analogía rápida

- `deleteMany({})` → vaciar la caja 📦

- `drop()` → destruir la caja 💥

## Buenas prácticas

Antes de borrar:

- ✅ Siempre revisar con `find()`

``` javascript

db.usuarios.find({ edad: { $lt: 18 } })

```
- ✅ Luego borrar

``` javascript

db.usuarios.deleteMany({ edad: { $lt: 18 } })

```

## Resumen

- `deleteOne()` → borra uno

- `deleteMany()` → borra varios

- `{}` → significa “todos”

- `deleteMany({})` → borra todo

- `drop()` → elimina la colección completa
