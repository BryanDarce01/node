# Statics

Los métodos estáticos no son una característica de MongoDB, sino de Mongoose (y de JavaScript).

## Ejemplo

Una fábrica de carros

Tenemos una clase:

``` javascript

class Car {
  constructor(color) {
    this.color = color;
  }
}

```

Creamos dos carros:

``` javascript

const car1 = new Car('rojo');
const car2 = new Car('azul');

```

Cada carro es un objeto diferente.

## 1. Métodos de instancia

Supongamos que cada carro puede tocar la bocina:

``` javascript

class Car {
  constructor(color) {
    this.color = color;
  }

  horn() {
    console.log('Beep!');
  }
}

```

Entonces:

``` javascript

car1.horn();
car2.horn();

```

- Aquí el método pertenece a cada objeto individual.

## 2. Métodos estáticos

Ahora imaginemos una operación que no depende de un carro específico, sino de la fábrica completa:

``` javascript

class Car {
  static totalCarsProduced() {
    console.log('1000 carros');
  }
}

```

No hacesmos:

``` javascript

car1.totalCarsProduced(); // ❌

```

Sino:

``` javascript

Car.totalCarsProduced(); // ✅

```

- Porque esa información pertenece a la fábrica, no a un carro.

### En Mongoose pasa lo mismo

Tenemos el modelo:

``` javascript

const Review = mongoose.model('Review', reviewSchema);

```

Y documentos:

``` javascript

{
  rating: 5,
  tour: 123
}

{
  rating: 4,
  tour: 123
}

```

- Cada uno es una review individual.

## 3. Método de instancia (`methods`)

Si definimos:

``` javascript

reviewSchema.methods.printRating = function() {
  console.log(this.rating);
};

```

Entonces:

``` javascript

const review = await Review.findById(id);

review.printRating();

```

Aquí:

``` javascript

this

```

es esa review:

``` javascript

{
  rating: 5,
  tour: 123
}

```

## 4. Método estático (statics)

Aquí definimos:

``` javascript

reviewSchema.statics.calcAverageRatings = async function(tourId) {
  ...
};

```

Y se usa así:

``` javascript

Review.calcAverageRatings(tourId);

```

Observemos que no hacemos:

``` javascript

review.calcAverageRatings();

```

- porque una review individual no conoce todas las demás.

### ¿Por qué es estático?

Porque calcular el promedio necesita ver muchas reviews:

``` javascript

[
  { rating: 5 },
  { rating: 4 },
  { rating: 3 }
]

```

Una sola review:

``` javascript

{ rating: 5 }

```

- no puede calcular el promedio de todas.

Por eso la operación pertenece al modelo completo:

``` javascript

Review.calcAverageRatings()

```
## Resumen

### ¿La operación trabaja con UN documento?

``` javascript

review.text = 'Excelente';
review.printRating();

```

- Usamos `methods`.

### ¿La operación trabaja con MUCHOS documentos o con toda la colección?

``` javascript

Review.aggregate(...)
Review.find(...)
Review.calcAverageRatings(...)

```

- Usamos `statics`.

**Los métodos normales (`methods`) pertenecen a un documento.**

**Los métodos estáticos (`statics`) pertenecen al modelo completo.**
