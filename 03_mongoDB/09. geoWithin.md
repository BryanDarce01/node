# `$geoWithin`

Es un operador geoespacial de MongoDB que permite responder preguntas como:

- ¿Qué documentos están dentro de esta área?

Por ejemplo:

- ¿Qué tours están dentro de 10 km de Ciudad de Guatemala?

- ¿Qué restaurantes están dentro de un polígono dibujado en un mapa?

- ¿Qué usuarios están dentro de cierta ciudad?

## 1. Primero necesitamoss un campo geoespacial

Ejemplo este esquema:

``` javascript

const tourSchema = new mongoose.Schema({
  startLocation: {
    type: {
      type: String,
      default: 'Point',
      enum: ['Point']
    },
    coordinates: [Number],
    address: String,
    description: String
  }
});

```

## 2. Y un índice geoespacial:

``` javascript

tourSchema.index({ startLocation: '2dsphere' });

```

Ejemplo

``` javascript

{
  name: 'The Forest Hiker',
  startLocation: {
    type: 'Point',
    coordinates: [-90.734, 14.558]
  }
}

```

Tiene que ser en este orden

``` javascript

coordinates: [longitud, latitud]

```

No:

``` javascript

[latitud, longitud] ❌

```

## ¿Qué hace `$geoWithin`?

Busca documentos cuya ubicación esté dentro de un área.

La sintaxis es:

``` javascript

Tour.find({
  startLocation: {
    $geoWithin: {
      ...
    }
  }
});

```

## Ejemplo: Dentro de un círculo

Supongamos:

Centro:

```

Latitud = 14.558
Longitud = -90.734

```

Radio:

```

10 km

```

Consulta:

``` javascript

Tour.find({
  startLocation: {
    $geoWithin: {
      $centerSphere: [
        [-90.734, 14.558],
        radioEnRadianes
      ]
    }
  }
});

```

## ¿Por qué radio en radianes?

MongoDB trabaja con una esfera (la Tierra).

Se convierte:

**Kilómetros**

``` javascript

const radius = distance / 6378.1;

```

- 6378.1 km = radio aproximado de la Tierra.

**Millas**

``` javascript

const radius = distance / 3963.2;

```

## Ejemplo completo

Queremos tours dentro de 50 km:

``` javascript

const distance = 50;

const radius = distance / 6378.1;

const tours = await Tour.find({
  startLocation: {
    $geoWithin: {
      $centerSphere: [
        [-90.734, 14.558],
        radius
      ]
    }
  }
});

```

MongoDB devolverá:

``` javascript

[
  Tour1,
  Tour2,
  Tour3
]

```

- si están dentro del círculo.

## Endpoint

``` javascript

/tours-within/:distance/center/:latlng/unit/:unit

```

Ejemplo:

``` javascript

/tours-within/50/center/14.558,-90.734/unit/km

```

## Controlador

``` javascript

exports.getToursWithin = async (req, res) => {

  const { distance, latlng, unit } = req.params;

  const [lat, lng] = latlng.split(',');

  const radius =
    unit === 'mi'
      ? distance / 3963.2
      : distance / 6378.1;

  const tours = await Tour.find({
    startLocation: {
      $geoWithin: {
        $centerSphere: [[lng, lat], radius]
      }
    }
  });

  res.status(200).json({
    status: 'success',
    results: tours.length,
    data: {
      data: tours
    }
  });
};

```

# Diferencia con `$geoNear`

## `$geoWithin`

Responde:

- ¿Cuáles están dentro de esta área?

Devuelve:

``` javascript

Tour A
Tour B
Tour C

```

No calcula distancias.

## `$geoNear`

Responde:

- ¿Qué tan lejos está cada documento?

Devuelve algo así:

``` javascript

[
  {
    name: 'Tour A',
    distance: 3.2
  },
  {
    name: 'Tour B',
    distance: 7.1
  }
]

```

y normalmente los ordena por cercanía.

## Resumen

`$geoWithin` encuentra documentos cuya ubicación esté dentro de una región.

Normalmente se usa con:

``` javascript

$centerSphere

```

y requiere:

1. Un campo GeoJSON:

``` javascript

startLocation: {
  type: 'Point',
  coordinates: [lng, lat]
}

```

2. Un índice:

``` javascript

tourSchema.index({ startLocation: '2dsphere' });

```

3. Una consulta:

``` javascript

Tour.find({
  startLocation: {
    $geoWithin: {
      $centerSphere: [[lng, lat], radius]
    }
  }
});

```

que responderá:

- Dame todos los tours que están dentro de X kilómetros desde este punto.
