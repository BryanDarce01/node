# ¿Qué hace el set?

Por ejemplo:

``` javascript

ratingsAverage: {
  type: Number,
  default: 4.5,
  min: [1, 'Rating must be above 1.0'],
  max: [5, 'Rating must be below 5.0'],
  set: val => Math.round(val * 10) / 10
}

```

Aquí estamos usando un **Setter** de Mongoose.

# ¿Qué es un Setter?

Un **setter** transforma el valor antes de guardarlo en MongoDB.

Algo así:

``` javascript

valor recibido
     ↓
setter
     ↓
valor guardado

```

## Ejemplo

El aggregate calcula:

``` javascript

4.666666666666

```

y hacemos:

``` javascript

Tour.findByIdAndUpdate(id, {
  ratingsAverage: 4.666666666666
});

```

Antes de guardarse ejecuta:

``` javascript

Math.round(val * 10) / 10

```

## Paso 1

``` javascript

4.666666 * 10

```

Resultado:

``` javascript

46.66666

```

## Paso 2

``` javascript

Math.round(46.66666)

```

Resultado:

``` javascript

47

```

##  Paso 3

``` javascript

47 / 10

```

Resultado:

``` javascript

4.7

```

Entonces MongoDB guarda:

``` javascript

ratingsAverage: 4.7

```

## ¿Por qué usar un **setter** aquí?

Porque los promedios suelen generar números enormes:

``` javascript

4.666666666666667

```

y para una calificación eso se ve feo.

Y es mejor para el usuario:

``` javascript

4.7

```

como en Amazon, Airbnb, Booking, etc.

## Un detalle importante

Los setters se ejecutan cuando el valor se asigna al documento.

Por ejemplo:

``` javascript

tour.ratingsAverage = 4.66666;

```

↓

Setter

↓

``` javascript

4.7

```

Por eso cuando el método:

``` javascript

calcAverageRatings()

```

actualiza:

``` javascript

ratingsAverage: stats[0].avgRating

```

- el setter automáticamente redondea el promedio antes de guardarlo.

## Resumen

Esto es un setter:

``` javascript

set: val => Math.round(val * 10) / 10

```

y sirve para redondear automáticamente el promedio de ratings a un decimal:

```

4.666666 → 4.7
4.21     → 4.2
4.25     → 4.3

```

sin tener que hacerlo manualmente cada vez que actualizamos el Tour.
