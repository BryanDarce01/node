# Mongoose 

Es una librería que nos ayuda a trabajar con MongoDB de forma más ordenada

- Es un ODM (Object Data Modeling)

## ¿Qué significa ODM?

Es como un “traductor” entre:

- JavaScript (objetos)

- MongoDB (documentos)

### Sin Mongoose

``` javascript

db.collection('usuarios').find()

```

### Con Mongoose

``` javascript

Usuario.find()

```

# 1. SCHEMA (lo más importante)

Es donde se define cómo deben ser los datos

``` javascript

const mongoose = require('mongoose')

const usuarioSchema = new mongoose.Schema({
  nombre: String,
  edad: Number,
  email: {
    type: String,
    required: true
  }
})

```

## ¿Para qué sirve?

- Define estructura

- Valida datos

- Puedes poner reglas

Aunque MongoDB es flexible, aquí nosotros le decimos orden

# 2. MODEL

Es la forma en que interactuamos con la base de datos

``` javascript

const Usuario = mongoose.model('Usuario', usuarioSchema)

```

### Hacemos esto

``` javascript

// Crear
await Usuario.create({ nombre: "Bryan", edad: 25 })

// Leer
const usuarios = await Usuario.find()

// Actualizar
await Usuario.updateOne({ nombre: "Bryan" }, { edad: 26 })

// Eliminar
await Usuario.deleteOne({ nombre: "Bryan" })

```
### Ventajas

- Estructura (schemas)

- Evita datos desordenados

# Validaciones

### 1. `required`

``` javascript

email: {
  type: String,
  required: [true, "El email es obligatorio"] // o solo true
}

```

- `true` → el campo es obligatorio

- `"mensaje"` → error personalizado

### ¿Qué pasa si no se manda?

Mongoose lanza error:

```

El email es obligatorio

```
### 2. `unique`

``` javascript

email: {
  type: String,
  unique: true
}

```

Significa:
- no pueden existir dos documentos con el mismo valor

#### Ejemplo

``` javascript

await Usuario.create({ email: "test@mail.com" })
await Usuario.create({ email: "test@mail.com" }) // ❌ error

```

## IMPORTANTE

- `unique` NO es una validación como `required`

- Es un índice en MongoDB

### ¿Qué significa eso?

- MongoDB evita duplicados a nivel de base de datos

- Pero el error no es tan “bonito” como `required`

Ejemplo

No viene con mensaje personalizado fácil

```
E11000 duplicate key error
```

### 3. `min` y `max`

``` javascript

edad: {
  type: Number,
  min: [18, "Muy joven"],
  max: [60, "Muy viejo"]
}

```

### 4. `match`

``` javascript

email: {
  type: String,
  match: [/^\S+@\S+\.\S+$/, "Email inválido"]
}

```

## Resumen

- `required: true` → obligatorio

- `required: [true, "mensaje"]` → obligatorio + mensaje

- `unique: true` → no duplicados (índice, no validación)

Todo se puede combinar
